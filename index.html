What is HTML?

HTML (HyperText Markup Language) represents the content and structure of a web page. It is the foundation every website is built on.

02

Elements

Elements are the building blocks of HTML. Most consist of an opening tag and a closing tag wrapping content.

<elementName>Content</elementName>
03

Void Elements

Void elements cannot have content and only have a start tag. Common examples:

<img>  <meta>  <link>  <br>
04

Attributes

Attributes provide additional information about an element or specify its behavior.

<a href="https://example.com">Link</a>
<elements>

Common HTML Elements

Headings

Six levels of headings indicate content importance. Lower number = higher importance.

<h1>Most Important</h1>
<h2>Second Level</h2>
<h3>Third Level</h3>
<h4>Fourth Level</h4>
<h5>Fifth Level</h5>
<h6>Least Important</h6>

Paragraph & Text

Use <p> for blocks of text, <em> for emphasis, and <strong> for urgency.

<p>Cats <em>love</em> lasagna.</p>

<p><strong>Important:</strong>
  Wear safety goggles.
</p>

Images

The <img> element embeds images. Always include src and alt attributes.

<img
  src="cat.jpg"
  alt="A tabby cat"
>

Links

Anchor elements create links. The href attribute specifies the destination.

<a href="https://example.com">
  Click here
</a>

Lists

Use <ul> for unordered (bulleted) and <ol> for ordered (numbered) lists.

<ul>
  <li>Catnip</li>
  <li>Laser pointer</li>
</ul>

<ol>
  <li>Flea treatment</li>
  <li>Thunder</li>
</ol>

Figure & Caption

<figure> groups images with <figcaption> for accessible captions.

<figure>
  <img src="cats.jpg"
    alt="Two kittens">
  <figcaption>
    Cats <strong>hate</strong>
    other cats.
  </figcaption>
</figure>

Semantic Layout

Semantic elements describe their meaning to both the browser and the developer.

<main>...main content...</main>
<section>...grouped content...</section>
<footer>...copyright...</footer>
<div>...generic container...</div>

Comments

Comments leave notes in your code for yourself and other developers. They are not rendered.

<!-- This is an HTML comment -->
<structure>

HTML Structure & Boilerplate

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <title>Page Title</title>
    <link rel="stylesheet"
      href="./styles.css" />
  </head>
  <body>
    <!-- Content goes here -->
    <script src="app.js"></script>
  </body>
</html>
DOCTYPE

Tells browsers which version of HTML you're using.

<html lang="en">

Root element of the document. lang sets the document language.

<head>

Contains metadata — info for browsers and search engines, not visible to users.

charset="utf-8"

UTF-8 encoding supports virtually all characters on the web.

<body>

All visible content lives here — headings, paragraphs, images, etc.

IDs & Classes

IDs are unique per document. Classes can be reused for grouping & styling.

<media>

Media & Multimedia

Audio

The <audio> element supports mp3, wav, and ogg formats. Use controls, loop, and muted boolean attributes to control playback.

<audio
  src="music.mp3"
  controls
  loop
  muted
></audio>

<!-- Multiple sources for browser compatibility -->
<audio controls>
  <source src="audio.ogg" type="audio/ogg" />
  <source src="audio.mp3" type="audio/mpeg" />
</audio>

Video

The <video> element supports mp4, ogg, and webm. The poster attribute shows a preview image before the video plays.

<video
  src="clip.mp4"
  controls
  loop
  muted
  poster="thumbnail.jpg"
  width="620"
></video>

iFrame (Inline Frame)

iFrame embeds external HTML content — including YouTube videos — directly in your page.

<iframe
  src="https://www.youtube.com/embed/VIDEO_ID"
  width="560"
  height="315"
  allowfullscreen
  title="YouTube video player"
></iframe>

Image Formats

Prefer modern formats over PNG/JPG for better performance:

  • WEBP — great compression, wide support
  • AVIF — best compression, growing support
  • SVG — infinite scalability, no quality loss

Paths, Links & Target Attributes

Absolute Path

A complete URL — includes protocol, domain, and file path. Use for external resources.

<a href="https://freecodecamp.org">
  Visit freeCodeCamp
</a>

Relative Path

Relative to the current file's location. Shorter and flexible for internal links.

<a href="about.html">About Page</a>
<link href="./styles.css" />
<img src="../images/cat.jpg" />

Target Attribute Values

Value Behavior
_self Opens in the current tab/window (default)
_blank Opens in a new tab or window
_parent Opens in the parent context (e.g., outside an iframe)
_top Opens in the topmost browsing context

Link States (CSS Pseudo-classes)

:link unvisited
:visited already seen
:hover cursor over
:focus keyboard focus
:active being clicked
<meta>

SEO & Open Graph

Meta Description

Provides a short page summary for search engines — impacts SEO rankings.

<meta
  name="description"
  content="Expert tips on gardening."
/>

Open Graph Tags

Controls how your page appears when shared on social media (Facebook, LinkedIn, etc.).

<meta property="og:title"
  content="My Site" />
<meta property="og:type"
  content="website" />
<meta property="og:image"
  content="preview.png" />
<meta property="og:url"
  content="https://mysite.com" />

&copy; → ©
